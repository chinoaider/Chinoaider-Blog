搭建一个Hexo博客

发表于 2023-04-12
  • Blue Forest
首页 Hexo 教程

本博客采用 Hexo shoka 主题 + GitHub + Verce 搭建 主演示为 win10 环境

# 准备工作

  • 安装 Node.js 环境
    • ​ 前往官网：https://nodejs.org/ ，演示环境为当前新版本 18.15.0 LTS
  • 注册 GitHub 账户
    • 前往官网：https://github.com/ ，注册账户
  • 安装 Git 终端
    • 前往官网： https://git-scm.com/downloads ，选择对应的系统文件进行下载
    • 如果没有特殊要求一路 next 默认安装即可

# 安装 Hexo

在 cmd 输入命令安装

# 创建 GitHub 仓库

点击左侧 New 按钮或者点击右上角 + 按钮 ，选择 New repository ，创建一个 <用户名>.github.io 的仓库

创建仓库的格式必须为： <用户名>.github.io

  • Description (optional) ：仓库描述（选填）
  • Public ：互联网上的任何人都可以看到这个存储库。你选择谁能做出承诺。（选择）
  • Add a README file ：在这里你可以为你的项目写一个长描述。学习更多的知识。（勾选）
  • 点击 Create repository 进行创建

# 配置 Git

  1. 查看配置文件
  1. 配置用户名和邮箱

# 生成配置 ssh 公钥

  1. 输入以下命令生成 ssh 公钥，用于计算机与 GitHub 连接，ssh 公钥文件路径在 C:\Users\username\.ssh\id_rsa.pub 打开全选复制 ssh 公钥
  1. 进入 GitHub 👉 点击头像 👉 选择 Settings 👉 选择 SSH and GPG keys 👉 点击 New SSH key
  • Title ：标题随便填
  • key ： 粘贴 ssh 公钥
  • 点击 Add SSH key 进行添加
  1. 输入命令测试连接

出现连接到账户信息，说明大功告成，至此完成准备工作。

# 初始化 Hexo 项目

  1. 创建一个用于存放项目的文件夹，打开 cmd 命令窗口，执行 hexo init 命令初始化项目
  1. 进入 your project ，输入 nmp i 安装相关依赖
  1. 初始化项目后，目录结构如下

node_modules ：依赖包

scaffolds ：模版文件夹，当新建文章时，Hexo 会根据 scaffold 来建立文件

source ：存放用户资源的地方

themes ：主题文件夹，默认安装 landscape 主题，可以安装新主题到 themes 目录

_config.yml ：网站的配置信息文件，可以在此配置大部分的参数

_config.landscape.yml ：主题的配置文件

package.json ：应用程序的信息。可以查看 hexo 的版本以及安装的扩展版本

  1. 输入以下命令，启动项目
  1. 运行后出现以下信息，按住 ctrl 鼠标点击访问，如果显示正常说明本地部署项目成功

# 将项目挂载到 GitHub

  1. 安装部署所需依赖
  1. 修改 _config.yml 文件，末尾处 username 改成你的用户名， repository 改成仓库名，主分支 main
  1. 将代码部署到 GitHub

现在任何人都可以访问到你的博客了 https://repository.github.io ，注意这里的 repository 是你的仓库名
