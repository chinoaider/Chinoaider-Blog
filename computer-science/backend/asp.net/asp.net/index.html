# Asp.Net 常用内置对象

# Respones

Response 对象用于将输出从服务器发送给客户端。这个对象有许多集合、方法和属性，常用于控制输出的内容和行为。

# Write

Response.Write 用于将指定的字符串写入输出

也可以在服务器端根据条件生成内容并将其发送到客户端浏览器

# Buffer

当 Response.Buffer 设置为 true 时（默认值），服务器会将页面输出缓冲起来，直到所有服务器脚本在当前页面上都被处理完毕，或者直到调用了 Flush 或 End 方法

缓冲处理可以确保在服务器端执行完所有脚本之前，不会向浏览器发送响应，确保页面的完整性和一致性

但是，如果页面包含大量脚本或长时间运行的操作，缓冲可能导致响应延迟，因为客户端无法立即看到部分数据。

# Expires

Response.Expires 设置在浏览器中的缓存存储过期时间（单位为分钟，默认值为 20 分钟）。如果不想页面被缓存，可以将其设置为 0

# Redirect

Response.Redirect 用于重定向到新 URL 的方法，常用于在页面处理过程中将用户导航到不同的页面或网址

~/ 表示从根目录开始， ./ 表示从当前目录开始， ../ 表示当前目录的上级目录

使用 Response.Redirect 时，有时会遇到 “正在中止线程”（ ThreadAbortException ）的异常。这通常是因为在页面处理过程中使用了 Response.Redirect 或 Response.End 等方法所导致

也可以使用 javascript 实现跳转

# ContentType

Response.ContentType 设置响应的内容类型，例如 text/html 、 application/json 等

# ContentEncoding

Response.ContentEncoding 设置响应的内容编码，例如 UTF-8

# Cookies

Response.Cookies 获取或设置发送到客户端的 Cookies 集合

# Request

Request 对象用于处理客户端发送到服务器的 HTTP 请求

# Browser

Request.Browser 用于获取有关请求客户端的浏览器功能的信息

# Form

Request.Form 用于获取通过 HTTP POST 方法提交的表单数据。它返回一个包含所有表单字段及其值的集合，这个集合可以通过字段名称来访问特定的表单数据

# QueryString

Request.QueryString 用于获取通过 HTTP GET 方法提交的表单数据。URL 将表单数据通过 GET 方法传递给服务器，表单数据会附加在 URL 的查询字符串（query string）中，格式为 ?key1=value1&key2=value2

注：因为 get 方法传输的格式为 ?key1=value1&key2=value2 ，如果传输内容有出现 & 符号，可能会导致 ” 错误认领 “ 的结果

解决方法

当你需要在 URL 中传递特殊字符（如空格、问号、等号等）时，可以使用 Server.UrlEncode() 这个方法将这些字符编码为 URL 安全的格式编码，再用 Server.UrlDecode() 这个方法将其解码为可读的格式

string encodedUrl = Server.UrlEncode("Hello World!");

string decodedUrl = Server.UrlDecode("Hello%20World%21");
