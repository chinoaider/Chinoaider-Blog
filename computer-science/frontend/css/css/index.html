# 简介

CSS+DIV 布局方式是一种前端开发中常用的布局方式，它的优点在于大大缩减页面代码，提高页面浏览速度，缩减带宽成本；结构清晰，容易被搜索引擎搜索到，天生优化了 SEO；缩短改版时间。以下是一些常用的布局代码例子，你可以参考它来实现你自己的布局。

# 圣杯布局

是一种三栏布局，其中两边的列宽度固定，中间的列宽度自适应。圣杯布局的特点是可以实现三栏布局，中间一栏最先加载、渲染出来。

# 优点

  • 两边固定宽度，中间自适应
  • 先加载中间的内容（因为两边是导航栏和广告栏，中间是内容区）

# 缺陷

  • 某一列内容溢出时，三栏不是等高的。（进行等高布局解决）
  • 中间栏的最小宽度不能小于左右两栏的宽度，否则左右两栏会掉到下一行。（设置 min-width 解决）

# 双飞翼布局

双飞翼布局是一种三栏布局，其中两边的列宽度固定，中间的列宽度自适应。双飞翼布局和圣杯布局类似，但是双飞翼布局通过设置中间栏的 margin 来实现左右两栏位置的保留。

# 优点

  • 双飞翼布局省去了很多 css，由于不使用定位，中间栏的最小宽度不再受限于左右两栏宽度。

# 缺陷

  • 双飞翼布局多加一层 dom 树节点，增加渲染树生成的计算量， dom 结构不如圣杯布局简单直观。
