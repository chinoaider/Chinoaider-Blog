# Minecraft 服务端搭建 ⛏

# 准备工作

演示系统：群晖 NAS7.1

下载套件：【Java21 OpenJDK】、【Git】、【Docker】

套件来源：【矿神源：https://spk7.imnks.com/ 】

其他资源：

Minecraft 官网 Java 版服务端 Jar 包：https://www.minecraft.net/zh-hans/download/server

MC 资源整合网站苦力怕：https://klpbbs.com

# Java 版本兼容

较新版本的游戏需要较新版本的 Java

某些版本的 Minecraft：Java 版服务器需要最低 Java 版本才能运行。以下是 Minecraft 服务器的 Java 版本要求：

  • Minecraft ≥ 1.20.5 和 1.21 需要 Java 21。
  • Minecraft ≥ 1.18 需要 Java 17。
  • Minecraft ≥ 1.12 需要 Java 8。

有关 Minecraft：Java 版的 Java 版本要求的完整列表，请访问教程：更新 Java 为什么要更新？

旧版本的 Minecraft 可以在较新的 Java 上运行，例如 Minecraft 1.16 可以在 Java 17 上运行，Minecraft 1.18 可以在 Java 20 上运行，等等。

下载对应的 Java 版本

# Docker

群晖社群里直接下载 Docker 套件

打开后会发现注册表无法访问，通常需要翻墙或者挂代理或在设置里添加国内镜像链接，我这里没找到有效镜像链接就直接用代理方式好了

# 开启代理服务器

  1. 代理方式，推荐 Clash，购买机场添加订阅链接后打开允许局域网连接，这个【7890】代理端口，鼠标移到开关上或者 【win+r】打开【cmd】 输入指令 【ipconfig】这个以太网【IPv4 地址 】就是代理地址，原理是开启后代理服务会映射到本地 IP

  1. 这里需要先把 Clash 这个软件的防火墙入站规则设置一下，把这个阻止 Clash 软件入站规则给禁用，或者直接关闭防火墙

  1. 把你的网关和端口填写到群晖的网络配置里的代理服务器，勾选通过代理服务器连接并应用即可

  1. 关闭重新打开 Docker 定位到注册表选项卡，你会发现镜像列表出现了

# MC 服务端搭建搭建流程

GitHub：https://github.com/itzg/docker-minecraft-server

  1. 在注册表输入：【itzg】 搜索，第一个就是【Java 版】Minecraft 服务端镜像，双击选择直接下载镜像

  1. 下载后会出现在你的映像选项卡里，选中后单击启动或直接双击

  1. 网络直接默认如图即可，下一步

  1. 配置环境变量

常规设置里其他选项不需要勾选默认后面也可以修改，直接点击高级设置

  • VERSION（Java 服务端版本）：默认 【LATEST】 最新版本），建议填写需要的版本不然以后更新的时候会直接覆盖旧版本，我这里填 1.21.5（现阶段最新版本）

  • EULA（最终用户许可协议）：默认为空或者没有，这里填 【TRUE】 （表示同意协议）后面才能一次运行内部署完成

  • GUI（权限）：【101】表示管理员权限

高级设置主要注意的环境变量就这三个，保存，下一步

  1. 配置端口

  • 容器端口：默认【25565】不可更改

  • 本地端口：【25565】端口如果被占用就填写没被占用的端口 ，【25575】远程控制端口

  • 类型：【TCP】Java 版只支持 TCP 协议

下一步

  1. 挂载文件夹

点击添加文件夹，直接选择要存放到哪个目录，装载路径【/data】，下一步，完成

  1. 配置完后会在容器里出现，默认自动启动等待项目下载运行完

单击详情，查看日志，出现以下地址端口就说明项目已经下载运行成功了

  1. 现在可以在电脑上打开 Minecraft 启动器开始游戏了，支持大家购买使用正版账号登陆，有正版账号的直接在 Xbox 或微软商店里直接下载启动游戏

没正版 Java 版账号的可以使用 PCL2 启动器，必须下载和服务端对应的游戏版本的，我这里是最新版本所以我下载 1.21.5 版本，直接在启动页面选择版本，启动游戏 PCL2 启动器会自动下载对应 Java 服务和所需依赖

记得修改【server.properties】文件关闭正版验证，不然无法进入游戏，文件通常在服务端根目录下

GitHub： https://github.com/Hex-Dragon/PCL2
