# 导入模组流程

这里是手机 Minecraft 游戏客户端的教程

如果有正版账号就更简单了，可以省略很多授权步骤，在 pc 端正版启动器 mc 存档文件是可以直接访问的，原理也是差不多，把服务器存档拉下来放到客户端下运行存档安装模组，测试到完美之后，再把安装好模组的存档上传到服务器上

# 下载模组

首先前往 【 https://klpbbs.com/ 】下载模组，也叫 BE 附加包，就是游戏中的行为包和资源包，下载的模组要和你的客户端对应

模组一般不兼容跨一个大版本如：客户端 1.9.x ❌ 模组 1.20.x （有的也兼容，很少）

模组一般可以跨小版本向下兼容如：客户端 1.21.1 ✅ 模组 1.21.2 (有些也不兼容)

# 更改存储位置

手机端在导入模组之前要先在游戏设置中找到存储这一行，把文件存储位置改成外部，这样运行生成游戏的时候存档就会生成到外部文件存储位置，导入的文件也会在外部文件存储位置

# MT 管理器

  1. 用 mt 管理器打开下载好的 BE 附加包，或定位到所在位置或用其他格式打开，找到 mc 游戏客户端打开方式

  • 后缀一般为 mcpack 和 mcaddon ，注意检查后缀名，不然会找不到 mc 游戏客户端打开方式

  • mcpack 需要用 mt 管理器打开 类型 👉 全部 👉 找到 mc 游戏客户端打开方式，打开后会直接导入 mc 游戏内

  1. 导入成功后创建一个新存档，或者编辑本地的存档，找到行为包这一行就能看到导入的模组，激活后进入游戏去测试你添加的模组是否兼容，测试到没问题之后再进行下一步

  1. 游戏默认情况下会生成在根目录的 data/data 下的对应文件夹中，一般情况下是无法访问的，这需要 root 权限，而改成外部文件存储位置后，就会生成在 /storage/emulated/0/Android/data/ 这个路径下，一般情况下也无法访问，前往 mt 管理器设置，打开请求 Shell 权限，这需要用 Shizuku 来授权，https://shizuku.rikka.app/

  1. 根据官方文档配置完后就可以访问到游戏存储文件夹了

  • behavior_packs ：行为包文件夹，用于修改游戏中实体的行为逻辑。
  • custom_skins ：自定义皮肤文件夹，存放玩家自定义的游戏角色皮肤。
  • development_... : 与游戏开发相关的文件夹，可能用于存储开发过程中的临时文件或自定义开发内容。
  • logs ： 日志文件夹，记录游戏运行过程中的各种信息，便于排查问题。
  • minecraftpe ：与《我的世界》基岩版 (Pocket Edition) 相关的文件夹。
  • minecraftWorlds ：存放游戏世界存档的文件夹，每个子文件夹对应一个游戏世界。
  • resource_packs ：资源包文件夹，用于修改游戏的纹理、声音等资源。
  • skin_packs ： 皮肤包文件夹，存放成套的角色皮肤。
  • world_templates ：世界模板文件夹，包含创建新游戏世界时可用的模板。

# 上传配置文件

  1. 进入 minecraftWorlds 继续进入下一级目录，会看到以下文件，只需要这四个文件，其中的 json 文件中已生成调用了模组

  1. 在上传前，如果怕存档出问题可以先备份一下服务器上的 【worlds】文件夹，然后先暂停 mc 服务端

  2. 将它们打包上传到服务端的存档路径下 ./worlds/Bedrock level/

  3. 将下载的模组后缀改成 zip，上传到服务端后解压，

  • 可以放到根目录下的 ./behavior_packs 文件夹中，

  • 也可以放到存档下的 ./worlds/Bedrock level/ behavior_packsxia 文件夹中，二者其中一个有这个模组就行

  • 接着重启服务端即可，这样就可以在游戏内用服务器方式进入游戏使用模组了
