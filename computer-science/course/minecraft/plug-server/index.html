# 插件服、模组服安装

  1. 先前往官网下载服务安装工具，【neoforge】和【forge】的安装方式大差不差，这里只演示【Forge】

forge：https://files.minecraftforge.net/

neoforge：https://projects.neoforged.net/

右键【installer】 复制下载链接，将前缀去掉，这个就是安装工具的 jar 包

  1. 打开 Ubnunt 后端命令窗口，定位到挂载 Forge 文件夹内，【ls】查看文件列表

输入以下命令进行安装服务

下载安装完成后，目录下会生成 【run.sh】 文件

如果没权限，输入以下命令赋予权限，再执行 【./run.sh】命令

如果出现 【sudo】 无效命令，输入以下命令安装 【sudo】

第一次运行 ./run.sh 会要求你同意 EULA 协议

用 nano 指令编辑 eula.txt 将 false 改成 true ，或者直接在 nas 服务器上打开文件修改

修改后 CTRL + X 保存退出

# Geyser

建议使用稳定版本，Java 和 Bedrock 互通

游玩前需要进行测试，测试下载的 Java 和 Bedrock 版本直接会不会出现不兼容现象

下载：https://geysermc.org/

# 可修复

以下问题可以通过 Hurricane 插件 / 模块解决。然而，这些问题需要修改 Java 服务器。

  • 由于 Java 和 Bedrock 之间的偏移差异导致竹子周围的运动问题。
  • Geyser 模拟的是 1.21.5 版本的客户端，因此 Java 服务器必须接受使用该版本的用户才能正常运行。这得益于 ViaVersion，它允许使用较新版本的 Java 版玩家加入运行较旧版本游戏的服务器。

# Floodgate 验证

  • 强制启用 Floodgate 验证

  • 前往下载所需依赖：https://modrinth.com/mod/floodgate?loader=neoforge&version=1.21.5

  • 配置 ./config/Geyser-NeoForge/config.yml 文件中设置 floodgate，否则基岩玩家会被要求使用 Java 账号登陆
