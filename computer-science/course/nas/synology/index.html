# 准备工作

# 硬件需求

需要一台 nas 小主机，可以是旧的电脑主机也可以自己 diy 一个，推荐蜗牛星际，适合新手入手学习使用，部分品牌机电脑或服务器可能会出现各种定制主板的不兼容毛病，建议采用正规 DIY 渠道配件。尽量不购买品牌机安装黑群晖（部分品牌会锁 BIOS 安装不了其它系统，或者设置非常麻烦），不需要无线网卡，网线直连，这样网络比较稳定

需要一个空 U 盘，这个 U 盘是作为黑群晖系统引导启动盘来使用，每次打开 nas 主机都会通过这个 U 盘来引导进入群晖系统，所以必须一直插 nas 主机在上面

需要至少一块机械硬盘，最好是两块，群晖双盘位 nas 安装系统时默认给你开了 shr 或者 Raid1，两块盘互为备份。不想要的话可以在系统装好后进入 “储存空间管理员” 删除储存空间重建 basic（无数据保护）储存池。一些比较重要数据比如照片什么的也可以备份在云盘，后面再说怎么把云盘当作备份盘来使用，如果有盘里文件最好先备份一下

# 兼容问题

自己 DIY 渠道配件 nas 主机的话可以问客服是什么型号是否兼容等问题，如果是自己的旧主机，则需要了解你的主板和 cpu 的型号是否兼容，请自行百度相关参数

举个例子，我的主机配置：

cpu：i5-3450

主板：爱国嘉 b75

内存：两条 8G

硬盘：西数 1T

ps: 在技嘉辱华前也没有觉得技嘉的东西有多好用，没办法很久以前买的了那时候还不是很懂被实体店坑的，在这里抵制爱国嘉所有产品，但真金白银买的东西不用又浪费，当作废物利用吧

爱国嘉 GA-B75M-D3V 芯片组或 北桥芯片 Intel B75 CPU 插槽 LGA 1155 支持 CPU 类型 支持 Core 三代 i7/i5/i3,Core 二代 i7/i5/i3 系列处理器，物理群晖不支持 9 代以上爱国嘉和微星主板，这里 B75 是支持的

CPU i5-3450 是 3 代酷睿，一般 4 代及以后处理器可选 918、920 引导，3 代及以前处理器可选 3615、3617 引导，或者都可试试，cpu 参数自行百度，还有一点，如果考虑功耗的话，推荐 J N D 开头的工控机 CUP（只有零售桌面型号，J N D 开头的工控机 CUP 功耗才低）

如果不知道自己 cpu 的年代可以点此连接查看：https://www.mydrivers.com/zhuanti/tianti/cpu/

RAID 控制卡是一种 磁盘阵列卡 ，它的核心就是 RAID 控制芯片 。 随着 RAID 技术的发展，现在的 RAID 控制卡不再局限提供 SCSI 一种磁盘接口，在 PC 机中常用的 IDE 和 SATA 接口 现在也可全面支持 RAID 技术了，而且在中低档 磁盘阵列 中应用非常广，特别是新兴的 SATA 接口的 RAID 控制卡。

群晖只支持软 RAID（并非主板的 RAID）和部分有驱动的硬件 RAID 卡，但是非常不建议硬件 RAID 卡因为官方也是软 RAID 这样可提供可视化报警。服务器改群晖需要注意群晖只支持部分硬件 RAID 卡，建议把阵列卡刷成 IT 直通模式这样兼容性会更好一些，请自行咨询服务器销售商确认 RAID 卡是否支持 IT 直通，群晖系统能否使用

# 开始教程

# 下载引导

前往巴西大佬主页下载：https://github.com/fbelavenuto/arpl/releases

这个 arpl.img 就是引导文件，先把这个下载下来

解压后会获得一个 arpl.img 引导文件，需要使用工具把它写到我们准备好的 U 盘里

这边推荐另一个引导文件下载的链接

# U 盘刷写工具

推荐：rufus

前往下载：https://rufus.ie/downloads/

设备选择你的 U 盘，引导类型选择你刚刚解压出来的 arpl.img 引导文件，点击开始等待写入完毕

# 载入引导

  1. 把刷写好的 U 盘接入要当 Nas 的机子，接上键盘，接上显示器，接上网线，开机
  2. 按 DEL（不同主板快捷键不一样）进入 BIOS 界面，设置 U 盘首选启动项（自行百度），按 F10 保存退出
  3. 重新进入时会看到这个界面，它会自动载入

等待引导加载配置

接着进入会这个页面，会出现一个 IP 地址

注：正常显示出 ip 地址就可以在同局域网内的其它电脑浏览器输入这个地址就可以进入编译选单，如果 ip 都没显示说明网卡没有正常被驱动请检查好网卡：具体看这里：安装引导过程中 nas 主机屏幕没出现局域网 ip

# 编译引导

使用另一台电脑打开浏览器，输入刚刚这个显示器上的 IP，端口是 7681 ，进入自编译界面，如果不想用电脑编译的也可以直接在屏幕输入 menu.sh 回车或按键盘的光标 “↑“再回车也可以直接 ARPL 菜单

这里选择第一项 Choose a model ，直接回车

这个列表就是我的处理器支持的群晖系统型号了，选择你要刷的群晖系统型号，我是 3 代处理器，这里我选择 DS3617xs 型号进行演示

注：一般 4 代及以后处理器可选 918 、 920 引导，3 代及以前处理器可选 3615 、 3617 引导，或者都可试试，如果你没有看到 918 或者想刷的型号说明你的处理器不支持那个型号的系统，

这里是让你选择系统版本，选中 Choose a Build Number 回车

选择你要刷的版本， 42962 是群晖 7.1 版本，选择回车

这里可以让你选择手动输入一个序列号，或者生成一个序列号，选择 Choose a serial number 直接敲回车

如果没有正版序列号的话就选择第一个生成随机序列号，第二个是手动输入序列号，我没有正版序列号这里我选择第一项，敲回车

接着选择 Build the loader ，进行编译引导

注：如果是十代处理器，选择 Addons 回车进入，选择 Add an Addon 回车，选择 i915 intel iGPU Drivers(10th Gen) 回车，在 params 参数处不输入任何内容，回车即可。再选择 Exit 返回上级菜单，继续选择 Build the loader ，进行编译引导（不是十代处理器省略此步，此操作是为了添加核显驱动）

等待引导下载、编译完成

完成后会自动跳转到菜单，选中 Boot the loader 回车

引导启动成功，记住这个型号和版本，后面需要根据这个来下载 DSM 补丁，我的是 DS3617xs 型号和 42962 版本

# 搜索设备

先在浏览器输入这个网址：https://finds.synology.com/ 搜索你的设备，如果在这个链接搜不到设备，尝试在路由后台查看有没有你的设备 IP 再进行搜索

在浏览器进入你的路由后台： 192.168.10.1 ，登陆进入后台（账号密码在路由器底部）

到路由器后台查看有没有对应的 IP（前面 Nas 主机屏幕出现的 IP），如果有的话，在浏览器打开

在浏览器输入你的设备 IP，加上端口 5000 搜索你的设备，我的设备 IP 是： 192.168.10.45:5000

注：到这一步有可能会出现无法搜索到设备的情况，或者识别不到硬盘等情况，具体请看故障排错：引导成功后搜索不到设备 IP 、未检测到硬盘

# 下载上传 DSM 补丁

成功搜索到设备后点击安装，会来到这个页面，选中从计算机手动上传.pat 文件，点击从 Synology 下载中心 下载 DSM 补丁

点击 所有下载 跳转到 DSM 所有版本下载页面

选择你的版本，我的是 DS3617xs 型号和 42962 版本，所有选这个 7.1.1-42962 点进去看有没有 DS3617xs 这个型号的 DSM 补丁

可以看到里面没有这个型号，那我们继续返回上一级继续往上一个个查找

直到找到你的型号的 .pat 文件为止，按 Ctrl+F 搜索文件名，看到这个 3617xs.pat 结尾这个就是了，要注意文件大小，这个文件是 300MB 左右，文件太小也不对，下载下来，其他型号同理，按照这个方法来找

浏览，选择刚才下载的 .pat 文件，下一步

选中复选框，点击继续

注：到这里可能会出现卡进度的情况，如果出现请看这里：黑群晖安装过程中报错卡进度

等待格盘完毕，自动重新启动

# 创建存储池和存储空间

好了，黑群晖系统成功安装，进入到主页，接下来开始创建存储池和存储空间

这里可以根据自己的需求组成 RAID，如果是 DSM7，那么强烈建议你使用 SHR，而且我这里只有一个盘，后期需要添加新的硬盘我可以通过 SHR 进行多盘拓展，点击下一步

RAID 0：连续以位或字节为单位分割数据，并行读 / 写于多个磁盘上，因此具有很高的数据传输率，但没有数据冗余。

RAID 1：通过磁盘数据镜像实现数据冗余，在成对的独立磁盘上产生互为备份的数据。当原始数据失效时，系统可以自动切换到镜像磁盘上读写。

RAID 2：数据以位或字节为单位分割后存入各磁盘，并使用汉明码来提供错误检查及恢复。这种编码技术需要多个磁盘存放校验信息，实施很复杂，因此很少使用。

RAID 3：将数据条块化分布于不同的硬盘上，使用简单的奇偶校验，并用单块磁盘存放奇偶校验信息。如果一块磁盘失效，可用奇偶盘及其他数据盘重建数据。对于随机数据来说，奇偶盘会成为写操作的瓶颈。

RAID 4：将数据以块或记录为单位分布于各磁盘上，一块磁盘专存校验信息。每次写操作都需要访问校验盘，所以校验盘成为写操作的瓶颈，因此很少使用。

RAID 5：为避免校验盘成为瓶颈，将数据、校验信息交叉存于各磁盘。RAID 3 与 RAID 5 相比，最主要的区别在于 RAID 3 每进行一次数据传输就需涉及到所有的阵列盘；而对于 RAID 5 来说，大部分数据传输只对一块磁盘操作，并可进行并行操作。在 RAID 5 中有 “写损失”，即每一次写操作将产生四个实际的读 / 写操作，其中两次读旧的数据及奇偶信息，两次写新的数据及奇偶信息。

RAID 6：与 RAID 5 相比，RAID 6 增加了第二个独立的校验信息块。两个独立的校验系统使用不同的算法，数据的可靠性非常高，即使两块磁盘同时失效也不会影响数据的使用。但 RAID 6 需要分配给校验信息更大的磁盘空间，相对于 RAID 5 有更大的 “写损失”，因此 “写性能” 非常差。较差的性能和复杂的实施方式使得 RAID 6 很少得到实际应用。

这里如果你有多块盘，想使设置 raid1 的话，就选择两个硬盘并设置 RAID 类型为 “镜像”，下一步，会弹出提示，不用管点击继续

# 故障排错

常见问题故障排错

# 安装引导过程中 nas 主机屏幕没出现局域网 IP

  1. 网卡没有正常被驱动请检查好网卡，在引导里面加入自定义的扩展驱动 extra.lzma 文件

方法：

用压缩软件打开 DSM.pat，提取 zImage 和 rd.gz

解压扩展驱动 extra.lzma 和 extra2.lzma

用 DiskGenius 打开 u 盘第二分区，替换上述四个文件（直接改引导镜像再写入也行）

插入新引导和新硬盘，正常安装测试

下载工具：DiskGenius – 正式版下载 | 免费下载

  1. 也可能是群晖内核不支持的网卡型号，可以尝试 6.1 或者 6.2 里面不同版本的引导

# 引导成功后搜索不到设备 IP

  1. 搜索的设备和 nas 设备不在同一个局域网中（网段不同），把设备和 nas 同样接到光猫上或者接到同一个路由器上，或者把路由改成桥接模式
  2. 物理群晖不支持 9 代以上技嘉和微星主板解决方法：只能更换主板或者套虚拟机
  3. 如果还是一样没有用，则可能是网段不同或者电脑防火墙限制导致，也可能是 kernel 崩溃掉了，可以看看这篇 关于群晖 ARPL 界面能出现 ip 但是使用 Synology Assistant 搜索不到 ip 问题 及解决方法 ，不行的话就得换型号重新引导了，把能试的都试一遍，再者就是主板不支持物理群晖，后续会出一个排错，或者另外的办法来安装。

具体可参考这几篇：群晖引导成功之后群晖助手 / Synology Assistant 搜索不到机器 ip 解决方法

# 未检测到硬盘

  1. 进入 BIOS，主板里的硬盘模式改为 ACHI， IDE 模式群晖系统是不认的
  2. 主板 SATA 开启了软 RAID 功能，导致占用了前置的所有 SATA 口，就算你还插有其它 SATA 卡也无法识别到硬盘。请尝试关掉主板上的软 RAID 功能

其他具体可参考这篇： 群晖提示 未在 xxxx 中检测到硬盘 找不到硬盘解决方法

# 黑群晖安装过程中报错卡进度

  1. 下载的 DSM 补丁版本不对应，版本型号一定要对应
  2. U 盘的 PID 和 VID 一定要对应，（ARPL 引导自动适配不用检查）。
  3. 拿到另一台电脑格式化一下硬盘，如果看到硬盘有被其它系统的引导分区也会报错，请删除分区重新建立成一个分区
  4. 一般就是以上 1 到 3 点的问题，如果还是有问题网上有很多玄学方法可以试试，比如改 BIOS，把 OS Type 改成 Windows8、启用 CSM、改 only Legacy 等等，是否有用未验证，具体自行百度，最直接就是换引导，也许是不兼容哪里出了毛病

具体可参考这篇：安装黑群晖 过程中 报错误 13 卡 8% 40% 52% 55%、56%、57%、58% 解决方法

注：安装失败后群晖助手显示可迁移，但实际上安装过程还是会格式化整个硬盘并不会保留数据。
