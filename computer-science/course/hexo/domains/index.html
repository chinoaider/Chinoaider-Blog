# Vercel 简介

Vercel 是一个开箱即用的网站托管平台，方便开发者快速部署自己的网站。它在全球都拥有 CND 节点，因此比 Github 官方自带的 github pages 更加稳定，访问速度更快。它还可以提供免费的前端应用托管及 Serverless Functions ，对开发者更加亲和与友好。

# Vercel 部署

  1. 前往官网：https://vercel.com ，注册 Vercel 账号，推荐使用 GitHub 账号授权登录。

  1. 登录后找到 GitHub 托管的 Hexo 博客仓库项目，直接 Import Git 仓库地址。

Project Name ：项目名称

Deploy ：点击部署

  1. 部署环节会自动执行，并生成访问域名，同时你可以看到应用的预览效果。

# 解析绑定域名

如果你还没有购买域名，可以跳过这一步

腾讯云、阿里云等都是不错的选择，但博主是白嫖党，就选择了 Freenom 注册域名，注意 Freenom 已经不支持注册免费域名了。由于我是使用 Freenom 注册的域名，没办法使用 @ 符解析主域名，所以我这里选择只解析一条二级域名。在腾讯云或者阿里云注册购买域名的朋友可以解析两条主机记录，分别是 www 和 @ ，在 Vercel 部署域名可以把 @ 的解析重定向至 www 。

以我的域名： chinoaider.com 为例进行说明

主机记录解释
www解析后的域名为 www.chinoaider.com
@直接解析主域名 chinoaider.tk
*泛解析，匹配其他所有域名 *.chinoaider.com
mail将域名解析为 mail.chinoaider.com ，通常用于解析邮箱服务器
二级如 abc.chinoaider.com ，填写 abc
手机网站如 m.chinoaider.com ，填写 m
显性 URL不支持泛解析（泛解析：将所有子域名解析到同一地址）

将域名指向云服务器，请选择 A ； 将域名指向另一个域名，请选择 CNAME ； 建立邮箱请选择 MX ，根据邮箱服务商提供的 MX 记录填写。

记录类型解释
A用来指定域名的 IPv4 地址（如 8.8.8.8），如果需要将域名指向一个 IP 地址，就需要添加 A 记录。
CNAME如果需要将域名指向另一个域名，再由另一个域名提供 IP 地址，就需要添加 CNAME 记录。
MX如果需要设置邮箱，让邮箱能收到邮件，就需要添加 MX 记录。
TXT在这里可以填写任何东西，长度限制 255。绝大多数的 TXT 记录是用来做 SPF 记录（反垃圾邮件）
NS域名服务器记录，如果需要将子域名交给其他 DNS 服务商解析，就需要添加 NS 记录。
AAAA用来指定主机名（或域名）对应的 IPv6 地址（例如：ff06:0:0:0:0:0:0:c3）记录。
SRV记录了哪台计算机提供了哪个服务。格式为：服务的名字、点、协议的类型，例如：_xmpp-server_tcp。
显性 URL从一个地址 301 重定向到另一个地址的时候，就需要添加显性 URL 记录（注：DNSPod 目前只支持 301 重定向）。
隐性 URL类似于显性 URL，区别在于隐性 URL 不会改变地址栏的域名。
  1. 点击 View Domains 前往添加域名

  1. 输入你的域名，点击 Add 添加域名

  1. 添加后 Vercel 对应域名下出现的三项值分别是：主机记录、记录类型和记录值。其中，主机记录是指你想要指定的子域名或根域名，例如 www 或 @ ；记录类型是指你想要添加的 DNS 记录类型，例如 A 、 CNAME 、 MX 等；记录值是指你想要添加的 DNS 记录值，例如 IP 地址或目标 URL 。

  1. 前往你购买域名的代理商，域名托管服务里，将域名指向你的网站

  1. 在浏览器地址栏输入你的域名访问

如果出现 css 样式加载失败，到 <root>/_config.yml ，把 url 改成你的域名即可

现在你已经可以通过自己的域名访问到 Hexo 博客了，Vercel 部署完成～🎉
