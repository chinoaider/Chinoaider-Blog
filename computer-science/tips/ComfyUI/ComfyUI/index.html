# ConfyUI 和 webUI 共用模型

操作步骤：

1. 在你的放置路径找到 ComfyUI-aki-v1.6→ComfyUI→extra_model_paths.yaml.example

2. 复制 extra_model_paths.yaml.example 文件删除后缀名.example（让文件名变成这样 extra_model_paths.yaml）

3. 用记事本打开 extra_model_paths.yaml 文件将里面的 D:/ConfyUI/models/heckpoints/ 改成 D:/sd-webui-aki-v4.10/（一定要是 / 不是 \ 还有路径这个不是既定的，要根据你所安装路径来只要最后一个是 sd-webui-aki-v4.10 就行）

# CLIPTextEncode

遇到的错误：

问题：当前使用的 checkpoint 并不包含 CLIP 或文本编码器， workflow 里某个节点（很可能是 CLIP Text Encode 或 Conditioning 相关的节点）仍然试图去连接一个 CLIP 输入，但这个输入是 None，因此报错。

解决方案：换一个包含 CLIP 的 checkpoint（比如 vae-ft-mse-840000-ema-pruned.ckpt ）

# VAEDecode

遇到的错误：

问题：输入图像（或潜空间张量）在 GPU 上，但 VAE 模型被加载到了 CPU 上

解决方案：使用 VAE Loader 节点显式加载 VAE 到 GPU

  1. 右键空白处 → Add Node → Loaders → Load VAE
  2. 选择一个 .vae.ckpt 文件（比如 sdxl-vae.safetensors 、 sdxl-vae.ckpt ）
  3. 把 Load VAE 的输出连接到 VAEDecode 节点的 vae 输入
